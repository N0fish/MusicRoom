### Сервисы:
- `api-gateway` — общий REST API, валидация, JWT-проверка, rate limit, Swagger; единая точка входа для фронта и мобильного клиента (по сабжекту API — основной контракт для клиентов).

- `auth-service` — аккаунты (email+пароль), выдача JWT; заготовки под соцлогин (Google/Facebook), как требуют «User» и «Mobile application» части.

- `playlist-service` — совместный редактор плейлистов (публичные/приватные; лицензии/приглашения; конкуренция правок).

- `vote-service` — «живое» голосование за треки в событии (видимость public/private + лицензии по приглашению/месту/времени).

- `realtime-service` — делегирование управления музыкой по устройствам (лицензии на уровне устройства).
WebSocket-хаб для realtime-уведомлений (перестановки в плейлисте, новые голоса, делегирование), журналирование действий клиента (по сабжекту — логировать платформу/девайс/версию).

### Хранилища: 
- `Postgres` (персистентные данные) + Redis (эфемерка и Pub/Sub).
Покрывает все обязательные сервисы из предмета.
